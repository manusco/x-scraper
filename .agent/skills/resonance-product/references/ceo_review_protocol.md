# CEO Review Protocol

> **Concept**: A high-rigor plan review that optimizes for ambition, leverage, and strategic alignment.

## 1. Scope Selection Modes

1.  **SCOPE EXPANSION (Cathedral)**: Envision the platonic ideal. Push scope UP. What would make this 10x better for 2x the effort?
2.  **SELECTIVE EXPANSION (Cherry-pick)**: Hold current scope as baseline, but surface expansion opportunities for the user to select individually.
3.  **HOLD SCOPE (Bulletproof)**: Acceptance of scope. Maximum rigor on architecture, edge cases, and observability.
4.  **SCOPE REDUCTION (Surgeon)**: Find the minimum viable version. Cut everything else.

## 2. CEO Cognitive Patterns

Apply these during the review:
1.  **Inversion Reflex**: For every "how do we win?", ask "what would make us fail?"
2.  **Focus as Subtraction**: The primary value-add is what to *not* do.
3.  **Proxy Skepticism**: Are our metrics serving users or have they become self-referential?
4.  **Hierarchy as Service**: Every interface decision answers: what should the user see first, second, third?
5.  **Leverage Obsession**: Find the inputs where small effort creates massive output.
6.  **Temporal Depth**: Think in 12-month arcs, not just this PR.

## 3. The 10x Product Check

Ask:
*   If the best engineer in the world had unlimited time and perfect taste, what would this system look like?
*   What adjacent 30-minute improvements would make this feature sing?
*   Does this plan move toward the 12-month ideal end state?

## 4. The Nuclear Scope Challenge

Before reviewing details:
*   **Premise Challenge**: Is this the right problem?
*   **Nothing Test**: What happens if we do nothing?
*   **Rebuild vs Refactor**: Is this rebuilding anything that already exists?
