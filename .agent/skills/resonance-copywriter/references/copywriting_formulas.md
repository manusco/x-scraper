# Copywriting Formulas & Page Frameworks

## 1. Core Formulas

### PAS (Problem-Agitate-Solve)
1. **Problem**: Name the pain clearly.
2. **Agitate**: Make the pain feel urgent and costly.
3. **Solve**: Present your solution as the relief.

### AIDA (Attention-Interest-Desire-Action)
1. **Attention**: Hook with a bold claim or question.
2. **Interest**: Expand with relevant specifics.
3. **Desire**: Connect to outcome they want.
4. **Action**: Clear CTA.

### BAB (Before-After-Bridge)
1. **Before**: Current painful state.
2. **After**: Desired outcome state.
3. **Bridge**: Your product is the bridge.

### 4 Ps (Promise-Picture-Proof-Push)
1. **Promise**: Lead with the outcome.
2. **Picture**: Paint the "after" vividly.
3. **Proof**: Evidence it works (data, testimonials).
4. **Push**: CTA with urgency.

## 2. Headline Formulas

| Formula | Example |
|---------|---------|
| `{Achieve outcome} without {pain point}` | "Ship faster without breaking things" |
| `The {category} for {audience}` | "The CRM for solo founders" |
| `Never {unpleasant event} again` | "Never miss a customer message again" |
| `{Question highlighting pain}` | "Tired of chasing invoices?" |
| `Stop {bad thing}. Start {good thing}.` | "Stop guessing. Start knowing." |
| `{Number} {things} that {outcome}` | "3 workflows that cut your deploy time in half" |
| `How {audience} {achieve outcome}` | "How remote teams stay aligned without meetings" |
| `What if {desirable scenario}?` | "What if every lead got a reply in 5 minutes?" |

## 3. CTA Formula Library

**Formula**: `[Action Verb] + [What They Get] + [Qualifier]`

### Strong CTAs
| CTA | When to use |
|-----|-------------|
| Start Free Trial | SaaS trial signup |
| Get [Specific Thing] | Lead magnet download |
| See [Product] in Action | Demo request |
| Create Your First [Thing] | Activation CTA |
| Download the Guide | Content offer |
| See Pricing for My Team | Pricing page |
| Book a Demo | Sales-assisted |
| Try It Free — No Card Needed | Trust-building |

### Weak CTAs (Avoid)
- Submit
- Sign Up
- Learn More
- Click Here
- Get Started (too vague — get started doing *what*?)

### Supporting CTA Text
Place below the button to reduce anxiety:
- "Free 14-day trial. No credit card."
- "Cancel anytime."
- "Join 5,000+ teams."
- "Takes 30 seconds."

## 4. Page-Specific Frameworks

### Homepage
- **Job**: Serve multiple audiences without being generic.
- **Headline**: Broadest value proposition. Outcome-driven.
- **Structure**: Hero → Social Proof → Problem/Pain → Solution → How It Works → Use Cases → Final CTA.
- **Rule**: Clear paths for different visitor intents (navigation, not a wall of text).

### Landing Page
- **Job**: Single message, single CTA. Complete argument on one page.
- **Headline**: Match exactly to ad/traffic source.
- **Structure**: Hero → Pain Agitation → Solution + Benefits → How It Works (3 steps) → Social Proof → Objection Handling → CTA.
- **Rule**: Remove navigation. One exit: the CTA.

### Pricing Page
- **Job**: Help visitors choose the right plan.
- **Headline**: "Simple pricing" or value-driven ("Invest in speed").
- **Structure**: Tier comparison table → Feature breakdown → FAQ → Social proof → Final CTA.
- **Rules**: Make recommended plan visually distinct. Address "which is right for me?" anxiety. Show annual/monthly toggle.

### Feature Page
- **Job**: Connect Feature → Benefit → Outcome.
- **Headline**: The outcome, not the feature name.
- **Structure**: Hero with outcome → How it works → Use cases → Integrations → CTA.
- **Rule**: Show, don't tell. Screenshots, GIFs, or interactive demos.

### About Page
- **Job**: Tell the story of why you exist.
- **Headline**: Mission or origin hook.
- **Structure**: Origin story → Mission/values → Team → Still include a CTA.
- **Rule**: Connect mission to customer benefit. This isn't a vanity page.

## 5. Section-Level Patterns

| Section | Purpose | Key Rule |
|---------|---------|----------|
| Social Proof | Build credibility | Use specific numbers + real names |
| Problem/Pain | Show you understand | Use customer language, not yours |
| Solution/Benefits | Connect to outcomes | 3-5 key benefits, not features |
| How It Works | Reduce complexity | 3-4 steps maximum |
| Objection Handling | FAQ + comparisons + guarantees | Answer "What's the catch?" |
| Final CTA | Recap value + repeat CTA | Risk reversal (guarantee, free trial) |
