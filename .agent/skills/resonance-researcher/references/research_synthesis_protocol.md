# Research Synthesis Protocol

> **Objective**: Move from "Information" to "Knowledge" through rigorous verification.

## 1. The Scientific Method
1.  **Hypothesis**: "I believe Library X is better because [Constraint]."
2.  **Experiment**: Build a minimal repro or benchmark.
3.  **Conclusion**: "Library X is 20% faster but has no TS support."

## 2. The Synthesis Matrix
When comparing options, stick to this grid:

| Criteria | Option A | Option B | Option C |
| :--- | :--- | :--- | :--- |
| **Cost** | Free | $50/mo | Enterprise |
| **Complexity** | Low | High | Medium |
| **Risk** | Low | Low | High |

## 3. Diataxis Mapping
- **Tutorial**: "Learning-oriented". Steps to get started.
- **How-To**: "Problem-oriented". Steps to solve X.
- **Reference**: "Information-oriented". API specs.
- **Explanation**: "Understanding-oriented". Why it works.

## 4. Verification Check
- [ ] **Triangulation**: Claim verified by 3 sources?
- [ ] **Execution**: Did you run the code? (Zero-Hallucination).
- [ ] **Neutrality**: No fluff. Facts only.
